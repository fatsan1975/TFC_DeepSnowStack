# DeepSnowStack / TFC Research Notes (Session 2026-02-16)

## 1) DeepSnowStack current architecture snapshot

- Entry point `DeepSnowStackMod` registers both common/client configs and attaches fog listeners to `NeoForge.EVENT_BUS` only on client dist.
- Config model in `DeepSnowConfig`:
  - Common accumulation tuning: thresholds (`heavyThreshold`, `extremeThreshold`), step caps, per-step probabilities, and vertical/survival toggles.
  - Client fog tuning: start/max intensity, curve, fog end/start targets and RGB blend targets.
- TFC precipitation bridge (`PrecipTfcCompat`):
  - Gets world climate via `WorldTracker` + `ClimateModel` and computes real intensity with `WeatherHelpers.calculateRealRainIntensity(...)`.
  - Snow gating uses `WeatherHelpers.getPrecipitationAt(...)` enum check (`Biome.Precipitation.SNOW`).
  - Multi-step stack count generated by intensity + configured probabilities/caps.
- Mixins:
  - `SnowLayerSupportMixin`: allows snow layer survival on top of full (8-layer) snow if enabled.
  - `WeatherHelpersPlaceSnowMixin`: intercepts `placeSnowOrSnowPile(level, pos)` and redirects full-layer behavior to vertical growth instead of sideways over-spread.
  - `WeatherHelpersSnowStackMixin`: intercepts TFC snow placement and applies multiple accumulation steps.
- Client fog (`DeepSnowFogClient`):
  - Snow-only fog + color blending based on real intensity.
  - Uses shader fog start/end and attempts SPHERE fog shape reflectively.

## 3) Critical compatibility finding for this repo

Using `javap` against the 4.0.19-beta jar shows `WeatherHelpers` signatures include:

- `private static boolean placeSnowOrSnowPile(ServerLevel, BlockPos)`
- `private static boolean placeSnowOrSnowPileAt(ServerLevel, BlockPos, BlockState)`

So on this TFC version, the `placeSnowOrSnowPileAt` target includes a **third `BlockState` parameter**. This means the older 2-arg inject descriptor is stale and needs updating (done in this session).
